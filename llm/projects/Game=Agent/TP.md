# 📝 大模型分布式核心：张量并行 (TP) 实战指南

## 1. 什么是 TP (Tensor Parallelism)？

**张量并行**是将模型的**每一层参数（权重矩阵）**直接切开，分给不同的 GPU 共同计算。

* **为什么要切？** Qwen3-8B 的参数量约为 80 亿。以 `bf16` 精度加载，仅权重就需要 **16GB** 显存。
* 
* 但在训练时，加上**梯度 (Gradients)** 和 **优化器状态 (Optimizer States)**，总显存需求会飙升到 **60GB~80GB** 以上。
* 
* **如果不切**：单张卡可能塞不下训练态的模型，直接 `Out of Memory (OOM)`。
* 
* **切了之后**：`tensor_model_parallel_size: 2`。
* 
* 这意味着 8B 的模型被横向切成了两半，GPU 0 拿一半，GPU 1 拿一半。


## 2. 你的 4 张 A800 是怎么分工的？

张卡形成了一个典型的**“训练-采样”分离架构**：

| 角色 | 硬件 (GPU ID) | 并行策略 | 任务说明 |
| --- | --- | --- | --- |
| **Learner** (训练) | **GPU 0, 1** | **TP2** | 负责反向传播、计算梯度、更新 Qwen3 的权重。 |

| **Rollout** (采样) | **GPU 2, 3** | **TP2** | 负责跑 `sglang` 推理，陪机器打斗地主，产生对局数据。 |

> “采用 TP2 是因为 8B 模型在训练时显存压力大。通过张量并行，我们将单层巨大的线性层（Linear Layer）切分到两块 A800 上，既解决了显存问题，又利用了 A800 之间极快的 NVLink 带宽进行同步。”


## 3. TP 的数学原理（以矩阵乘法为例）

想象模型里有一个巨大的矩阵乘法 $Y = XW$：

1. **行切分/列切分**：我们将权重矩阵 $W$ 切成 $W_1$ 和 $W_2$
1. 
2. **并行计算**：GPU 0 计算 $XW_1$，GPU 1 计算 $XW_2$。
1. 
3. **All-Reduce 通信**：两张卡算完后，通过 **NVLink** 交换一下结果，把两边的输出加起来，得到最终的 $Y$。

---

## 4. 关键配置参数对号入座

在你的 `config.yaml` 中，这些参数决定了 TP 的表现：

* **`tensor_model_parallel_size: 2`**
* 
* 核心配置。决定了模型被切成几份。


* **`sequence_parallel: true`**
* 
* **进阶点**：这不仅切了参数，还把 LayerNorm 和 Dropout 沿着序列长度方向切了，进一步省显存。


* **`attention_backend: flash`**
* 
* **配合 TP 使用**：FlashAttention 极大地减少了 TP 通信时的中间激活值显存占用。



## 5. 面试高频追问 (Q&A)

**Q: 为什么不选 TP4，把 4 张卡都用来训练？**

**通信开销**：TP 级数越高，每层之后的通信次数和延迟就越高。TP2 在单机内平衡了显存和速度。
**吞吐瓶颈**：强化学习（RL）需要大量的采样数据。如果 4 张卡都用来训练（Learner），推理（Rollout）就没卡了。采用 2+2 分离架构，可以让 Learner 在更新梯度时，Rollout 异步产生新对局，实现**流水线并行**，整体效率更高。



**Q: TP 和 DP (Data Parallelism) 有什么区别？**

* **A**:
* 
* **DP** 是每张卡都放一个**完整模型**，大家喂不同的数据（Batch）。
* 
* **TP** 是每张卡放**模型的一部分**。
* 
* **结论**：因为 8B 模型太大了，单卡塞不下完整的训练状态，所以必须先上 TP，再考虑 DP。




“在我的实验中，由于 A800 拥有 **80GB 显存** 和 **NVLink**

TP2 是最高效的选择。它保证了我们在处理斗地主长序列时，既有足够的显存 Buffer 容纳 `256` 的 Global Batch Size，又能保持极低的模型内并行延迟。”

TP (Tensor Parallelism) —— 实战中开启 (Value: 2)

--tensor-model-parallel-size 2

将 Qwen3-8B 的每一层线性层（Linear Layer）“纵向”劈开，一份参数存 GPU 0，一份存 GPU 1。

解决的问题：显存空间 (Memory) + 算力负载 (Compute)。8B 模型训练时，优化器状态和梯度非常占显存，TP2 成功把单卡压力减半。

强调 A800 的 NVLink。TP 会引入频繁的 All-Reduce 通信，但在 A800 高带宽下，TP2 的效率远高于 PP（流水线并行）。

这两个词是你面试中的“定海神针”。搞懂了它们，你就真正理解了大模型是如何在多张显卡上“合体”计算的。

## 线性层 (Linear Layer) —— 模型的“大脑细胞”

线性层是大模型中最基础、也是最消耗计算量的组件。在 Transformer 架构（如你的 Qwen3）中，它无处不在。

* **它的本质**：就是一个巨大的**矩阵乘法**。
* 
公式通常写作：

$$Y = X \cdot W + b$$


* $X$：输入的信号（比如“斗地主”的当前牌局信息）。
* 
* $W$：**权重矩阵**（模型学习到的知识）。
* 
* $Y$：输出的结果。


* **为什么在并行中重要？**：Qwen3-8B 的每一层都有非常宽的线性层（比如 Hidden Size 为 4096）。这个 $W$ 矩阵太大了，单张显卡算起来太吃力。
* 
* 当开启 **TP2** 时，你实际上是把这个大矩阵 $W$ **竖着劈成了两半**。GPU 0 存左半边，GPU 1 存右半边。

## 2. All-Reduce 通信 —— 显卡间的“对答案”

当你把线性层劈开后，问题来了：GPU 0 算出了一半结果，GPU 1 算出了另一半，**但谁都没有完整的最终答案**。这时候就需要 **All-Reduce**。

* **定义**：All-Reduce 是一种集体通信原语。它的目标是：让参与计算的所有 GPU 都能获得**所有卡计算结果的总和**。
* 
* **动作拆解**：
* 
1. **Reduce（求和）**：各个 GPU 把自己算出来的那部分结果发出来，大家加在一起。
1. 
2. **Broadcast（广播）**：把加出来的最终总和，再分发给每一个 GPU。

* **结果**：执行完 All-Reduce 后，GPU 0 和 GPU 1 手里都拿到了完整的 $Y$。

## 3. 它们是怎么结合在一起的？

面试官会问：“你的 TP2 是在什么时候发生通信的？”

你可以这样描述这个**“切开-计算-合体”**的过程：

1. **切分**：Qwen3 的一个线性层权重 $W$ 被切分为 $W_1$ 和 $W_2$。
1. 
2. **局部计算**：
1. 
* GPU 0 计算 $Z_1 = X \cdot W_1$
* 
* GPU 1 计算 $Z_2 = X \cdot W_2$

3. **All-Reduce 合体**：
1. 
* GPU 0 和 GPU 1 通过 **NVLink** 交换 $Z_1$ 和 $Z_2$ 并求和。
* 
* $$Y = Z_1 + Z_2$$

4. **同步输出**：现在 GPU 0 和 GPU 1 都有了完整的 $Y$，可以继续往下跑下一层了。

### 

因为在大模型里，**线性层是产生 All-Reduce 通信的源头**。

* 模型层数越多（Qwen3 有 36 层），线性层就越多。
* 
* 线性层越多，All-Reduce 就越频繁。
* 
* All-Reduce 越频繁，对 **NVLink**（那条高速公路）的依赖就越重。

### 

**“如果 All-Reduce 变慢了，会对训练产生什么影响？”**

> **Ans**： “会导致 **GPU 计算空转**。因为线性层是串行的，如果 All-Reduce 没完成，下一层就拿不到完整的输入，GPU 只能在那里等着，这会直接拉低模型的 **MFU（计算利用率）**。”

hidden_size,4096,模型每一层特征向量的维度（即“宽度”）。
