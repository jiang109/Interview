**流水线并行 (Pipeline Parallelism, PP)** 是大规模深度学习模型训练中的一种核心技术。

当一个模型的参数量大到连单张显卡（GPU）的显存都装不下时，流水线并行就派上用场了。

简单来说，它的核心逻辑是：**“横向切分模型，纵向接力计算。”**

---

### 1. 基本原理：切蛋糕模型

如果把一个巨大的深度学习网络看作一根长长的管道，流水线并行就是将这个管道**按层（Layers）**切割成若干段.

每一段交给一块不同的 GPU 负责。

* **模型切分**：假设模型有 16 层，我们有 4 块 GPU。我们可以让 GPU 0 负责 1-4 层，GPU 1 负责 5-8 层，以此类推。
* 
* **顺序接力**：在前向传播时，数据先在 GPU 0 计算，结果传给 GPU 1，再传给 GPU 2... 直到最后一块 GPU 计算出损失（Loss）。
* 
* 反向传播时，梯度则按相反的方向传回。

---

### 2. 核心挑战：流水线气泡 (Pipeline Bubble)

朴素的流水线并行存在一个致命问题：**设备闲置**。

如果像接力赛一样，只有等前一个人跑完，下一个人才能出发，那么在任意时刻，其实只有一块 GPU 在工作。其他 GPU 都在“干等”数据传过来。这些等待的时间被称为**流水线气泡**。

为了解决这个问题，研究者引入了 **微批次 (Micro-batching)** 的概念：

* 将一个大批次（Mini-batch）的数据进一步切分成多个更小的“微批次”。
* 
* 当第一块 GPU 处理完第一个微批次并传给第二块时，它立刻开始处理第二个微批次，而不是坐等结果返回。

---

### 主流方案

#### PipeDream (1F1B 模式)

这是目前最常用的策略（如 Megatron-LM 所采用）。系统进入稳定状态后，每块 GPU 处理完一个前向传播，紧接着就处理一个反向传播。

* **优点**：及时释放了不再需要的中间变量，**显存占用更低**，流水线更加平滑。

---

### 4. 流水线并行的优缺点

| 特点 | 描述 |
| --- | --- |
| **优点：显存节省** | 解决了单机无法加载超大模型的问题，各卡只需存储模型的一部分。 |

| **优点：通信量小** | 只在切分点的层与层之间传输张量，通信开销通常低于数据并行。 |

| **缺点：气泡开销** | 无论如何优化，流水线头尾总是存在一定的设备空闲。 |

| **缺点：负载均衡** | 如果每块 GPU 负责的层计算量不等，最慢的那块卡会拖慢全队速度。 |

---

### 总结

流水线并行通过**层间拆分**解决了“模型太大”的问题，而微批次调度（如 1F1B）则解决了“效率太低”的问题。在训练千亿或万亿级参数模型（如 GPT-4）时，通常会结合**张量并行 (TP)** 和**数据并行 (DP)** 组成 3D 并行方案。
